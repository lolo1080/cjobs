Site title Site title is the field for the PPC system name.
Base site url This is the base URL of your site (or service). Be aware to include the trailing slash (/).
For example:  http://www.mysite.com/ppc/.
Login url This is the URL to login script. Be aware to include the trailing slash (/).
For example:  http://www.mysite.com/ppc/admin/.
Sign Up url This is the URL to sign up script.
For example:  http://www.mysite.com/ppc/admin/m_registration.php.
E-mail confirmation url This is the URL to e-mail confirmation script.
For example:  http://www.mysite.com/ppc/admin/m_email_confirm.php.
Search area url This is the URL to main search page. Be aware to include the trailing slash (/).
For example:  http://www.mysite.com/ppc/search/.
Search url This is the URL to search script.
For example:  http://www.mysite.com/ppc/search/search.php.
XML search url This is the URL to XML search script.
For example:  http://www.mysite.com/ppc/search/xmlsearch.php.
Image url This is the URL to images. Be aware to include the trailing slash (/).
For example:  http://www.mysite.com/ppc/admin/images/.
Extra admin protection Extra admin protection is the field where your computer IP address could be entered. Here you can indicate an IP address to restrict access to the Admin Area only from indicated IP address.
Currency symbol Currency symbol is the field to type in the symbol of the currency to be used by PPC system to make payments.
For example:  $, £, ¥ etc.
Start balance Start balance is field to insert the value of initial balance a new Member will be provided with after signing-up.
For example:  5.
Minimum bid Minimum bid is the field to fill in the minimum cost of the listing. In case an Advertiser will try to place a listing with a cost less than the value indicated in this field the system will give a warning. The listing is not going to be added until a Member fill in the cost equal or higher than the value indicated in this field.
For example:  0.02.
Amount of listings The field to set a number of listings to be displayed on the Search results page.
For example:  20.
Amount Sponsored links The field to set a number of sponsored links to be displayed on the Search results page.
For example:  5.
Frame/window target The pull down menu defining how to open a site where the Member will taken upon execution of a click on links.
  • No traget - means the site where the Member has been taken upon execution of a click on links is opened in the same window (target="_self").
  • New Window - means the site where the Member has been taken upon execution of a click on links is opened in a new window (target="_blank").
  • Entire Window - means the site where the Member has been taken upon execution of a click on links is opened in the same window in top frame (target="_top").
Display xml sites name The pull-down menu defining either to display or not in Search results which XML Feed has been used to receive a listing.
For example:  If you select "Yes" then each listing with money earning will display the name of XML Feed where it was originated. For example: Results from Ah-ha being used for the results arrived from Ah-ha. If a listing is originated by the PPC system database, it will be displayed as Results from Our Database.
Display the bid amount The pull-down menu defining either to display or not a listing bid in Search results.
For example:  If you select "Yes" each listing with money earning will display its bid (so called Show Bid). Cost to advertiser: 0.01.
Cache results The pull-down menu defining either to include cache value or not into the results being originated by XML Feed. In case a cache value is included certain amount of disk space will be required, while the search process will accelerate.
Catch all keyword The pull-down menu defining either allow or not the use of "Catchall listings" listings by Members. The "Catchall listings" listings display Membrer's listing in search for keywords that have no bids on them.
Maximum number of "catch all" listings The field to fill in the maximum number of "catch all" listings that will be displayed in Search results page.
For example:  3.
Display rank The pull-down menu defining the layout of listing display.
  • Nothing - means no listings are displayed in a simple list mode.
    For example:  
         Listing_1
         Listing_2
         Etc.
  • Numbers - means that a listing is displayed in a numbered list mode.
    For example:  
        1. Listing_1
        2. Listing_2
        3. Etc.
  • Bullets - means that a listing is displayed in a bulleted list mode.
    For example:  
        · Listing_1
        · Listing_2
        · Etc.
Cache reset The field to fill in the time of cache cleaning. You can define the number of days after which the cache is cleaned automatically. The field value is supposed to be written in days.
For example:  3.
Advertiser bids The pull-down menu defining either the listings originated by the PPC system database are always first to be displayed among Search results regardless of listing bid or not. "Yes" - always are first in search results; "No" - are mixed with Listings from XML feeds by bid amount.
I feel lucky The pull-down menu defining either to display or not in Search results the button "I Feel Lucky". By clicking on this button, the Member will be automatically redirected to the random URL in the Search Results.
Maximum title length The field to fill in the maximum length of listing title. Use the range of 1 to 255 to indicate this value. This parameter defines the maximum title length to be filled in by Advertisers into their listings. In case Advertiser would try to type in the listing title with a number of characters exceeding a value assigned in this field, the system will report an error message.
For example:  100.
Maximum description length The field to fill in the maximum length of listing description. This parameter must be more then 1. This parameter defines the maximum size of the description field to be used by Advertisers in their listings. In case Advertiser would try to type in the listing description with a number of characters exceeding the value assigned in this field, the system will report an error message.
For example:  255.
Display field "Rank" You can enable/disable the field "Rank" into Sponsored links table in Member area.
SP Max increment Is the increment added to highest bid to set the Bid for top
Admin authorize Should all listings submissions be approved by you first? The pull-down menu defining whether the Advertiser's listings should undergo the Administrator's approval. As a result of &qout;Yes&qout; option selection each added Advertiser's list should wait for Administrator's approval. Until getting, an approval the Advertiser's listing is not displayed on the Search results page. An Administrator is entitled to approve or reject a listing. A Member will be informed about approval/rejection of his listing via e-mail. If you select &qout;No&qout; the listings are approved automatically.
Multiple signup The pull-down menu defining either the Member is allowed or not to sign up for more than one account using various login names. If you select "Yes", the Member is allowed to sign up for another account using different login name. If this setting is set to "No", the member is allowed to sign up for the PPC system membership only once. E-mail address are used for protection against repetitive sign-ups.
Confirm e-mail address The pull-down menu defining either to execute or not the e-mail address genuineness verification before confirmation of Member's account registration. If you select "Yes", then the first step of the sign up procedure a Member should type in his e-mail address. The letter containing the link enabling the next step of sign up procedure will be sent to the typed in e-mail address. The template of such letter is located in "Mail templates" > "Sign Up confirmation". Using such a feature Administrator can make sure that e-mail address really exists, and to get exact information about Member's e-mail address, which is necessary for a feedback purpose as well as to notify Members about some PPC system events.
Member approved The pull-down menu defining whether all Members signing up for account are subject to Administrator's approval. Administrator is entitled to approve or reject Member's access to the PPC system. If you select "Yes", Member's access to the PPC system will be suspended pending Administrator decision. If you select "No", access right will be given to Member automatically.
XML status by default The pull-down menu defining whether an XML Status will be available to each member upon signing up for account or not. The PPC system uses an approach where Affiliate has to the XML Status, while Administrator is entitled to approve (or reject) such a request. If you select an option "Yes", then Member can use the XML Status immediately after signing up to an account. He is enabled to use immediately the XML Feed PPC in his Affiliates program. If you select an option "No", then upon a completion of sign up procedure the XML Status is not available and to enable it you have to apply to Administrator from your account. Administrator can view the list of Members applied for use of XML Status clicking on "Submissions" > "XML Feed" in Admin Area.
Use full country name The pull-down menu defining whether the full name of the country where a search or a click was originated from have to be added to the statistics or not.
For example:  if you select an option "Yes", then the full name of the country is reflected in statistics. If the search has been originated from the United States, then in the column Country the will be United States. If you select an option "No", the short name of the country will appear in the list. So doing in the row reporting the search originated from the United States the symbols US will appear.
The log of xml search site The pull-down menu defining whether to display or not in Search results a detailed time statistics regarding search, grabbing, parsing on each XML Feed. If you select an option "Yes", then the table with debugging information will be displayed at the top of Search results page.
Affiliate program The pull-down menu defining whether Affiliates Program is going to be used in PPC system or not. Affiliates Program enables Affiliates to earn some money through searches and clicks. If you select an option "Yes", that means Affiliates Program is active and Members can earn some money clicking on links with earnings. If you select an option "No", that means Affiliates Program is disabled, and Members cannot earn any money through searches and clicks.
REMEMBER If this option is set to "NO", then despite any settings of PPC system concerning Affiliates, they will be not able to earn any money.
Affiliate payment system The pull-down menu defining type of affiliate payment system should be used. The two types of payment: a fixed one "Pay per click-trough" and "Pay Percentage of bid on click-through".
  • Pay per click trough - if you select this type of payment, the Affiliates will pay some fixed amount in US dollars or in conventional units regardless of listing bid has been clicked on.
  • Pay percentage of bid on click-through - in case you select this payment type, Affiliates will pay interest being deducted from Advertiser Bids of Internal Database Listings and XML Feeds listings.
Affiliates payment kind The field to fill in the amount to pay affiliates for click. If you chose "Pay per click-through" above, put a value in dollar (1 = $1, 10 = $10). If you chose "Pay percentage of bid on click-through" above, put a value in percent (1 = 1%, 10 = 10%).
Default referral percent It is the settings for your Referral program. Referral program means that members can invite users to you through their Referral Urls. Referral percent is the percent which will receive member from all earnings reffered by him user.
REMEMBER this setting is global for all users but you also can personally set up referral percent for any user. Click "Users" at top of page, then choose a user and click "Edit this referral configuration" and enter his personally referral percent.
Monthly affiliate program The pull-down menu defining whether Monthly Affiliate Program is going to be used in PPC system or not. The basic idea is each affiliate will earn on the number of children he's able to do rather then from the performed searches on PPC system each month. If you select an option "Yes", that means Monthly Affiliates Program is active and Members can earn some money. If you select an option "No", that means Monthly Affiliates Program is disabled, and Members cannot earn any money each month.
REMEMBER If this option is set to "NO", then despite any settings of PPC system concerning Monthly Affiliate Program, they will be not able to earn any money.
Affiliate will earn for new child Each affiliate will earn on the number of children he's able to do on PPC.
For example:  1;0.5;0.25;0.25. Each affiliate will earn $1 for each child, $0.50 for each grandson, $0.25 for each great grandson, $0.25 for each great grandson son.
Percentage for each following month Affiliate will earn percentage from he's children for each following month.
For example:  10. Users: PPC system, Gianluca, Ciro. Step 1) Gianluca is an affiliate of PPC system, when Ciro becomes a child of Gianluca: Gianluca earns $1 ($1 - "Affiliate will earn for new child" parameter) + $0.10 ($0.10 - 10% "Percentage for each following month" parameter) for each following month. Step 2) Ciro becomes an affiliate of PPC sytem, when Mike becomes a child of Ciro: Ciro earns $1 (as father) + $0.10 for each following month and Gianluca earns $0.50 (as grandfather) + $0.05 for each following month. Step 3) Mike becomes an affiliate of PPC sytem, when Tomas becomes a child of Mike: Mike earns $1 (as father) + $0.10 for each following month and Ciro earns $0.50 (as grandfather) + $0.05 for each following month and Gianluca earns $0.25 (as great grandfather) + $0.025 for each following month.
Affiliate activation limit When affilate reaches the "activation limit" his monthly affliate status changes to "active".
For example:  2. PPC system will pay really the affiliates when each of them becomes an active affiliate. The status of "active" affiliate is reached when the affiliate himself produces searches and clicks for a certain an amount ($2 for example) and maintain this status each month. When a new affiliate register himself his status obviously is "Not active", the system starts to count his earned money click (searches) and when he reaches the "activation limit" (the $2 above) his status changes to "active". Obviously if Member doesn't match the "activation limit" conditions during the month, on the 1st day next month searches/clicks for "Monthly affiliate program" will be lost.
